JavaScript Basics

JavaScript is a cross-platform, object-oriented scripting language. JavaScript is extremely popular for a variety of reasons. It is a small and ligjtweight language maximum flexibility for developers to take it in a bunch of different directions. JavaScript lives inside a bost environment (a web browser or Node server), it can be connected to the objects of these environment to provide programmatic control over them.

  • Variable Declaration JavaScript variables are containers for storing data values - imagine a cup you fill with coffe, the cup holds the coffe, a variable holds a value. All JavaScript variables must be identified with unique names. These unique names are called identifiers. var x
  • Variable Assignment Assignment operators assign values to JavaScript variables - our cup can now have coffee poured in it, giving our variable a value to hold. The = Assignment operator assigns a value to a variable. var x = 10;
  • Data Types Data types are an important concept; to be able to operate on variables toy need to know the data type. There are six data types that are JavaScript primitives; Boolean - true or false; null - null aka nothing; Numbers - 42 or 3.141592; String = "Coding Dojo Rocks!"; Array - [1,'Coding',2,'Dojo']; and Objects - {first_name: 'Jane', last_name: 'Doe'}
  • Conditionals When you write code, you want to perform different actions for different decisions - hitting different code blocks based on values or conditions that have been met. You can use conditionals statements in your code to accomplish this. There are the following conditionals statements: if a specified condition is true, do this code in our block; else if to specify a new condition to test, if the firts condition is false; else we execute this block of code;
  • Loops There are many different kinds of in every programming language, but they all essentially do the same thing: they will repeat an action some number of times. Imagine you have to run a mile, well you run arround the track four times and you stop. Thats a loop!
  • Functions Functions are an encaspulation of a code block. When we call our function this will run that code block. Think of it as a list of instructions. As an example imagine we are putting together a desk from ikea, we open up the instruction manual and get started, first we screw the legs to the table top; next we place the table the right way up. Done! We finished our instructions. Sadly there are a ton more tables to do so lets call our function over and over and over again.
